<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → protected routes | Unauthenticated requests must not reach app pages (proxy.ts) |
| route handler/RSC → Neon | User-scoped reads must not cross user boundaries (D-03) |
| client form → /api/auth/* | Untrusted form input crosses to auth endpoints |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-06 | Elevation of Privilege | src/lib/db/queries.ts | mitigate | Single and(eq(userId), eq(id)) on every user-scoped query (D-03 truth-condition); documented comment; no chained .where() (Pitfall 1). Verified by tests/idor.test.ts. |
| T-1-IDOR-404 | Information Disclosure | cross-user resource access | mitigate | Helper returns null → caller calls notFound() → 404 not 403 (D-09); existence never revealed. |
| T-1-PROXY | Elevation of Privilege | proxy.ts | mitigate | File named proxy.ts not middleware.ts (Pitfall 2); blanket redirect for non-public paths (D-08); session.id required. |
| T-1-XSS | Tampering | auth forms / error-banner | mitigate | CSP header from Plan 01 (D-13); React escapes interpolated copy; error strings sourced from copy.ts constants, not user input. |
| T-1-UI-ENUM | Information Disclosure | signup gating + login error | mitigate | signup notFound() when disabled (no "user exists" leak); login error from copy.ts is the single generic string (D-07). |
</threat_model>

# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation (auth screens).
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope

This contract covers only the screens delivered in Phase 1:

| Route | Screen | When Shown |
|-------|--------|------------|
| `/login` | Login form (email + password) | Always |
| `/signup` | Signup form (email + password + confirm) | Only when `SIGNUP_ENABLED=true`; otherwise 404/redirect |

No dashboard, no nav, no training content. These are the first screens a user sees and establish the visual language for all subsequent phases.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding |
| sm | 8px | Input padding vertical, field gap within a group |
| md | 16px | Default element spacing, input padding horizontal |
| lg | 24px | Section padding, gap between form fields |
| xl | 32px | Gap between form header and first field |
| 2xl | 48px | Touch target minimum (interactive controls) |
| 3xl | 64px | Card vertical padding on mobile |

Exceptions:
- **Touch targets:** All interactive controls (buttons, links) are minimum 48×48px hit area (source: additional_context). Visual size may be smaller with transparent padding applied.
- **Input height:** 48px minimum to satisfy touch target requirement and iOS tap accuracy.

## Typography

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Error messages, helper text, link labels |
| Label | 16px | 500 (medium) | 1.25 | Form field labels (matches input to prevent reflow) |
| Heading | 24px | 600 (semibold) | 1.2 | Page title ("Sign in to Pace", "Create account") |
| Display | 28px | 700 (bold) | 1.1 | Reserved for app logo / wordmark only |

**Input font size: 16px minimum.** This is a hard requirement — iOS Safari auto-zooms on input focus when font-size is below 16px (source: REQUIREMENTS.md PWA-05, additional_context).

Weights used: 400 and 600 only (two weights). Medium (500) on labels is permitted if shadcn's default component uses it; executor must not introduce a third custom weight.

## Color

Dark background palette. High-contrast for bright outdoor conditions (source: additional_context — "Dark background preferred, high-contrast for bright conditions").

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090b` (zinc-950) | Page background, full-bleed surface |
| Secondary (30%) | `#18181b` (zinc-900) | Auth card / form container surface |
| Accent (10%) | `#ffffff` (white) | Primary CTA button only |
| Border | `#27272a` (zinc-800) | Input borders, card outline |
| Text primary | `#fafafa` (zinc-50) | Headings, labels, input value text |
| Text muted | `#a1a1aa` (zinc-400) | Helper text, placeholder, "Don't have an account?" links |
| Destructive | `#ef4444` (red-500) | Error messages and error border states only |

Accent reserved for: **primary CTA button only** ("Sign in", "Create account"). No other element uses white as a filled background.

Error red is the only semantic color in scope for Phase 1. It appears on: inline field error text, field border when invalid, and the page-level error banner (rate limit / server error).

## Interaction Contract

### No hover-required interactions
All information and actions are reachable by tap. No tooltips, no dropdown-on-hover, no hidden-until-hover states. (Source: REQUIREMENTS.md PWA-04, additional_context.)

### Input attributes (required on all form inputs)
| Input | `type` | `inputmode` | `autocomplete` |
|-------|--------|-------------|----------------|
| Email | `email` | `email` | `email` |
| Password | `password` | (default) | `current-password` (login) / `new-password` (signup) |
| Confirm password | `password` | (default) | `new-password` |

### Focus state
Visible focus ring on all interactive elements. Use `ring-2 ring-white ring-offset-2 ring-offset-zinc-950` or shadcn's default focus-visible style.

### Loading state
Submit button shows a loading spinner (Lucide `Loader2` with `animate-spin`) and `aria-busy="true"` while the auth request is in flight. Button text changes from "Sign in" to "Signing in…" (no icon swap — text change only, spinner alongside).

### Error display timing
Errors appear inline after submission attempt — never on blur. One error at a time per field. Page-level errors (rate limit, server error) appear above the submit button inside the card.

## States per Screen

### /login

| State | What the user sees |
|-------|-------------------|
| Default | Email field, password field, "Sign in" button, "Don't have an account? Sign up" link (only when `SIGNUP_ENABLED=true`) |
| Loading | Button disabled, "Signing in…" + spinner |
| Field error | Red border on field(s), inline error text below field |
| Auth error | Page-level error banner: "Invalid email or password" |
| Rate limited | Page-level error banner: "Too many attempts. Try again in a few minutes." |
| Server error | Page-level error banner: "Something went wrong. Please try again." |

### /signup

| State | What the user sees |
|-------|-------------------|
| Default (enabled) | Email field, password field, confirm password field, "Create account" button |
| Default (disabled) | 404 page or redirect to /login |
| Loading | Button disabled, "Creating account…" + spinner |
| Field error | Red border on field(s), inline error text below field |
| Password mismatch | Inline error on confirm field: "Passwords don't match" |
| Registration closed | Page-level error: "Registration is not open." |
| Server error | Page-level error banner: "Something went wrong. Please try again." |

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Login page heading | "Sign in to Pace" | default |
| Signup page heading | "Create your account" | default |
| Login CTA (primary) | "Sign in" | AUTH-02 (log in action) |
| Login CTA (loading) | "Signing in…" | default |
| Signup CTA (primary) | "Create account" | AUTH-01 |
| Signup CTA (loading) | "Creating account…" | default |
| Auth error (invalid credentials) | "Invalid email or password" | D-07 (CONTEXT.md) — generic, no enumeration |
| Auth error (rate limited) | "Too many attempts. Try again in a few minutes." | AUTH-04 / D-10 — no timing info disclosed |
| Auth error (server) | "Something went wrong. Please try again." | default |
| Auth error (registration closed) | "Registration is not open." | D-11 (CONTEXT.md) |
| Password mismatch | "Passwords don't match" | default |
| Email invalid format | "Enter a valid email address" | default |
| Password too short | "Password must be at least 8 characters" | default |
| Link: login → signup | "Don't have an account? Sign up" | default |
| Link: signup → login | "Already have an account? Sign in" | default |
| Empty state | n/a — auth forms have no empty data state | — |
| Wordmark | "Pace" | CLAUDE.md project name |

**Anti-enumeration rule:** All auth errors use the same wording regardless of whether the email exists. The only error that references a specific field is a client-side format validation error (not a server response). (Source: AUTH-04, D-07.)

**Destructive actions in Phase 1:** None. Logout is a POST action with no confirmation required (session is stateless; user can re-authenticate immediately).

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Form fields have associated labels | `<label htmlFor>` or shadcn `<Label>` with `htmlFor` |
| Error messages linked to fields | `aria-describedby` pointing to error `<p id>` |
| Page-level error announced | `role="alert"` on error container |
| Button loading state | `aria-busy="true"` + `disabled` attribute |
| Focus visible on all interactive elements | `focus-visible:ring-2` via Tailwind |
| Color is not the only error indicator | Red border + error text (not border alone) |
| Password field label is "Password" not "pwd" | Full word labels only |

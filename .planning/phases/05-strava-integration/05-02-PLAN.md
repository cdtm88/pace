<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/strava/callback | Untrusted code/state/scope params arrive from Strava redirect |
| App → Strava API | Outbound; bearer token attached; 429/401 are adversarial-by-rate states |
| Server Action → iron-session cookie | CSRF state written/read here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Spoofing | OAuth callback CSRF | mitigate | state must equal session.pending_strava_state; state deleted + saved before token exchange (single-use, D-01) |
| T-05-06 | Elevation of Privilege | Scope escalation / wrong scope | mitigate | callback rejects unless granted scope includes activity:read (D-02) |
| T-05-07 | Information Disclosure | Token written/logged in plaintext | mitigate | both tokens encrypted via encryptToken before upsert; no console.log of token values |
| T-05-08 | Tampering | 401 from Strava → silent retry with stale token | mitigate | getValidAccessToken throws STRAVA_401; refreshStravaAction returns "disconnected" — prompt reconnect, no silent retry (D-07) |
| T-05-09 | Denial of Service | Strava 429 blocks user | accept/mitigate | exponential backoff (1s/2s/4s); after exhaustion surface "rate_limited" retry state, do not loop forever (D-14) |
| T-05-10 | Spoofing | Replayed/used auth code | mitigate | state deleted before exchange — a second callback fails state check cleanly (Pitfall 4) |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages in this plan — N/A |
</threat_model>

## Spacing Scale

Multiples of 4 only. Inherits scale established in Phase 1.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding |
| sm | 8px | Gap between chart bars, button group gaps |
| md | 16px | Card internal padding (--card-spacing default) |
| lg | 24px | Section separation on dashboard |
| xl | 32px | Vertical gap between major dashboard sections |
| 2xl | 48px | Page top padding (pt-16 = 64px, established in dashboard) |
| 3xl | 64px | Page top offset (existing: pt-16) |

**Exceptions:**
- Touch targets: all interactive controls must be ≥48×48px. Readiness tap buttons use `h-12` (48px). Strava Refresh button and Disconnect link must also reach 48px tap area — use `min-h-12` or explicit padding. PWA on-bike constraint (from ROADMAP Phase 6 PWA-03 — applied here proactively for Strava section buttons).
- "Connect with Strava" official SVG button: render at its natural branded size (approximately 193×48px per Strava guidelines). Do not scale down below 48px height.

## States Per Screen

### Strava Section — States

| State | Trigger | What User Sees |
|-------|---------|----------------|
| Disconnected | No stravaConnection in DB | Official "Connect with Strava" SVG button |
| Connecting | OAuth redirect pending | User navigates away to Strava; no in-page spinner needed (browser navigation) |
| Callback success | /api/strava/callback success | Redirect to /dashboard?strava_connected=1 — show brief success indicator via COPY.STRAVA_CONNECTED_SUCCESS toast/status |
| Scope denied | Callback lacks activity:read | Redirect to /dashboard?strava_error=scope_denied → ErrorBanner: COPY.STRAVA_ERROR_SCOPE_DENIED |
| Connected | stravaConnection exists | "Connected as {athleteName}" + Refresh + Disconnect |
| Refreshing | Refresh button clicked | Refresh button: disabled + Loader2 spinner (aria-busy="true") |
| 429 error | Strava API returns 429 after 3 retries | ErrorBanner + "Tap to retry" Button (variant="outline") below the banner |
| Disconnected (after disconnect) | disconnect Server Action succeeds | Returns to disconnected state; no confirmation modal |

### TSS Chart — States

| State | Trigger | What User Sees |
|-------|---------|----------------|
| Empty | No matched sessions | Chart frame renders at 300px height with centered text label (COPY.CHART_TSS_EMPTY_LABEL). No axis lines, no bars. |
| Populated | ≥1 matched session with TSS | BarChart with orange bars, X-axis week labels (e.g. "Jun 9"), Y-axis TSS values |
| Loading | SSR fetch in progress | Card renders server-side; no client-side loading state needed (RSC) |

## Destructive Actions

| Action | Approach |
|--------|----------|
| Disconnect Strava | Two-step inline confirmation. Clicking "Disconnect" replaces the button with inline text (`STRAVA_DISCONNECT_CONFIRM`) + two buttons: "Yes, disconnect" (variant="destructive") and "Cancel" (variant="ghost"). No modal. No page navigation. On confirm: Server Action fires, card returns to disconnected state. |

**Rationale:** No modal avoids a portaling implementation; inline confirm matches existing app interaction patterns (no modal infrastructure exists). The action is reversible (user can reconnect), so a simple inline confirm is sufficient.

## Interaction Patterns

### OAuth Connect Flow
1. User taps the "Connect with Strava" SVG image (wrapped in a `<form action={connectStravaAction}>` submit or a Server Action-triggered redirect).
2. Server Action: store `crypto.randomUUID()` as `session.pending_strava_state`, then `redirect()` to Strava authorization URL.
3. Strava redirects to `/api/strava/callback?code=...&state=...`.
4. Callback Route Handler validates state, exchanges code for tokens, encrypts tokens, stores in DB, redirects to `/dashboard?strava_connected=1`.
5. Dashboard reads query param and surfaces `STRAVA_CONNECTED_SUCCESS` via an inline success message (not a toast — no toast library installed). Remove param from URL after display.

### Refresh Flow
1. User taps "Refresh" button.
2. Button enters disabled + aria-busy="true" + Loader2 spinner state.
3. Server Action: proactive token refresh check (D-07: expiresAt - 600s), fetch 30 activities, run match algorithm, update DB.
4. On success: card re-renders with updated connected state (Server Component revalidation).
5. On 429 (after 3 retries): ErrorBanner + retry button visible below card.

### Retry Flow (429)
1. ErrorBanner shows `STRAVA_ERROR_RATE_LIMITED`.
2. "Tap to retry" Button (variant="outline") appears below ErrorBanner.
3. Tap re-invokes the same refresh Server Action.
4. If successful: ErrorBanner dismissed, card shows connected state.

## Accessibility Contract

- `<img src="/connect-with-strava.svg" alt="Connect with Strava" />` — descriptive alt text required by Strava brand guidelines.
- Refresh button: `aria-busy="true"` when loading; `aria-label="Refresh Strava activities"` if button shows only icon.
- ErrorBanner: `role="alert"` (already implemented in existing component).
- TSS chart: add `aria-label="Weekly training load bar chart, 6-week rolling window"` on `<ResponsiveContainer>` wrapper div.
- Disconnect confirm state: manage focus to first confirm button when inline confirm replaces the disconnect button.
- All interactive controls in the Strava section: min touch target 48×48px (PWA-03 constraint applied proactively).
- No hover-only interactions (PWA-04): all states reachable by tap.

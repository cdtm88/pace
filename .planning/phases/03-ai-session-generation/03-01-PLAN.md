<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI model → server | Claude JSON output is untrusted input crossing into the app; must be schema- and safety-validated before any DB write |
| authenticated user → rate limiter | A logged-in user could attempt to exceed the daily generation cap; the limiter key must be the cookie-derived userId, not client-supplied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | AI output → DB write (GeneratedSessionSchema) | mitigate | `GeneratedSessionSchema.safeParse` rejects malformed structure / out-of-bounds powerFraction before any insert (this plan defines the schema; Plan 02 wires the parse) |
| T-03-02 | Tampering | Physiologically unsafe session (validateSessionSafety) | mitigate | Deterministic safety gate caps powerFraction ≤ 1.5, ≤ 3 consecutive work blocks, ≥ 2 blocks, ≤ 4h — independent of the AI and of Zod (defense-in-depth) |
| T-03-03 | Denial of Service | Runaway AI spend (generationLimiter) | mitigate | `generationLimiter` 10/24h sliding window keyed on userId (cookie-derived); blocks before the Anthropic call in Plan 02 |
| T-03-04 | Elevation of Privilege | Cross-user session read (findLatestSessionByUserId) | mitigate | Query filters by `userId` from iron-session only; no client-supplied id; returns null for no-match |
| T-03-SC | Tampering | npm install (`@anthropic-ai/sdk`) | accept | Deferred to Plan 02 where the install occurs; RESEARCH.md Package Legitimacy Audit verdict: official Anthropic SDK, SUS-by-automation only, Approved (no [SLOP]) |
</threat_model>

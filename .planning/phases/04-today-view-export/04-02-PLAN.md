## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client (HTTP GET) → /session/[id] RSC | `[id]` URL segment is untrusted; userId read server-side from iron-session |
| RSC → SessionDetail client component | only the user's own session row + derived TSS/intensity are passed; rawJson is never forwarded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Elevation of Privilege | /session/[id] IDOR | mitigate | `findTrainingSession(ironSession.id, id)` with `and()` guard; `notFound()` (404 not 403) on null |
| T-04-08 | Information Disclosure | session row exposure to client | mitigate | RSC passes title/notes/blocks/derived TSS only; `rawJson` debug field is not forwarded as a prop |
| T-04-09 | Information Disclosure | resource existence via redirect timing | accept | auth (`!ironSession.id → redirect /login`) runs before the session fetch, so unauthenticated users never reach the IDOR branch |
| T-04-SC | Tampering | npm installs | mitigate | No new packages introduced (RESEARCH §Package Legitimacy Audit: none); slopcheck N/A |
</threat_model>
